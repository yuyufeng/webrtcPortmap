WebRTC PortMap
为什么 原理 信令服务器 Agent 客户端 ← 返回控制台
为 AI CLI 而生 · 持久远程终端

AI 在内网那台机器上跑，
你在哪都能接回去

你的 AI 编程 CLI（Claude Code、Codex、aider…）跑在公司工作站或家里的台式机上，正在执行一个长任务。人却出门了。webrtc-portmap 给那台机器一个 P2P 持久终端：用手机或笔记本随时接回去，断线重连不重置、AI 进程从不中断。

😖 它解决什么痛点 ⚙️ 工作原理 ① 信令服务器 ② Agent ③ 客户端
THE PAIN · 痛点

把 AI CLI 跑在远端，最难受的是这三件事

尤其当 AI 正在跑一个几十分钟的任务、而你必须离开屏幕的时候。

🚧

① 那台机器在 NAT 后面

公司/家里的机器没有公网 IP，被防火墙挡着。想从外面连进去跑/看 AI CLI，要么搭跳板机、要么开端口转发——麻烦又不安全。

📵

② 一断线，AI 会话就没了

SSH/普通终端一旦网络抖动或合上笔记本，会话断开、AI 进程被挂起或杀掉，几十分钟的上下文和输出全丢，只能从头再来。

🔀

③ 想换设备接着用

在工位上启动了任务，想在沙发上用手机、或换台笔记本接着同一个会话看进度、继续输入——传统方案做不到无缝接管。

THE FIX · 解法

持久终端：断线，但进程从不中断

终端进程挂在 Agent 上，独立于网络连接。你断开时它继续跑、输出写进环形缓冲；你重连时先回放历史、再接实时流。

在 工位电脑 操作
↺ 重连 · 回放历史输出 1.2KB（断线期间的全部内容）…
$ claude # 你的 AI CLI 跑在内网 Agent 上
● 分析仓库结构… done
● 运行测试 7/12 通过…
● 编辑 cmd/signaling/main.go
● 生成补丁 ▍
🔌
你的设备断开了
Agent 上的 shell 仍在运行 · 输出持续写入环形缓冲(256KB)
🔌

无需公网 IP

WebRTC P2P 直接打洞建连，打不通时自动回退内嵌 TURN 中转。Agent 主动外联，不用在内网开任何入站端口。

♻️

断线不重置

断开只解除输出回调、不杀进程；重连用 AttachWithReplay 先回放快照再接实时流，回放在前实时在后，不漏不重。

📱

换设备无缝接管

手机扫码 / 浏览器打开 / CLI -term，接进同一个会话，cwd、历史、运行中的 AI 任务原样还在。

HOW IT WORKS · 原理

它是怎么把内网那台机器接出来的

三个角色：公网信令服务器牵线，客户端与内网 Agent 之间走 P2P 加密直连。

客户端 信令服务器 Agent P2P 数据流
🛰️

信令服务器

公网常驻。负责信令交换、账户登录、内嵌 Web 控制台与可选 TURN 中转。终端/端口数据不经过它。

🖥️

Agent

内网常驻。预配置可访问端口列表，带 owner-hash 归属用户，可选内嵌持久终端。

💻

客户端

浏览器控制台直接用，或 CLI client 把远端端口映射到本地、进交互终端。

STEP 1 · 信令服务器

① 部署公网信令服务器

它是整个系统唯一需要公网的部分，自带 Web 控制台与可选 TURN 中转。

1

编译二进制

仓库根目录运行 buildserver.bat（Windows）或 ./build.sh，构建脚本会自动 go mod tidy 并下载 xterm 资源嵌入二进制。

2

最简启动（仅信令 + Web，纯 STUN 直连）

server — 最简
# 监听 8443，数据持久化到 data/signaling.json
$ ./bin/signaling -addr 0.0.0.0:8443 -data data/signaling.json
3

启用内嵌 TURN 中转（直连失败时回退，推荐）

server — 含 TURN + 管理员
$ ./bin/signaling -addr 0.0.0.0:8443 -data data/signaling.json \
    -turn-enabled -turn-public-ip <公网IP> \
    -turn-secret <固定随机串> -admin-config admins.json
⚠️ 必读：-turn-public-ip 必须是客户端可达的公网 IP，否则 relay 候选不可用；-turn-secret 要固定，留空会每次重启随机生成、导致旧凭据失效。
4

放行防火墙端口

端口协议用途
8443TCP信令 + Web 控制台（随 -addr 调整）
3478UDPTURN 中继（必须）
3478TCPTURN over TCP（建议，UDP 被封时回退）
5

打开控制台并注册

浏览器访问 http://<公网IP>:8443/ 注册账号、登录。登录后在页面上能拿到你的 用户 hash（user hash）—— 下一步启动 Agent 要用它做归属。

💡 Linux 生产部署：用 deploy/ 里的 systemd 方案，sudo ./install.sh ./signaling-linux-amd64 一键装好开机自启，配置在 /etc/webrtc-portmap/signaling.env。
STEP 2 · AGENT

② 在内网机器启动 Agent

Agent 主动连到信令服务器并上报能力；之后等待客户端通过 P2P 接入。

🔌

端口映射型

把内网服务（如本机 8080 的 http）暴露给授权用户访问。

agent — 端口
$ ./bin/agent -id myagent -name "我的客户端" \
    -owner-hash <user_hash> -password <本地密码> \
    -signal http://host:8443 \
    -ports ports.json
🤖

内嵌终端型（跑 AI CLI 的机器选这个）

开一个持久 shell，在里面起你的 AI CLI；断线重连不重置，回放历史输出、进程不重启。

agent — 终端
$ ./bin/agent -id myagent \
    -owner-hash <user_hash> -password <本地密码> \
    -signal http://host:8443 \
    -terminal -terminal-shell powershell
🤖 典型链路：在跑 AI CLI 的内网机器上以 -terminal 启动 Agent → 在它开出的 shell 里运行你的 AI CLI（claude / aider …）→ 从任意设备用浏览器「🖥️ 远程终端」或 client -term 接进去。断网、合盖、换设备都不打断 AI 任务。
关键参数说明
-id / -nameAgent 唯一标识 / 显示名
-owner-hashWeb 控制台拿到的用户 hash，决定归属（必填）
-password本地鉴权密码，DataChannel 建立后挑战-响应（必填）
-signal信令服务器地址
-terminal启用内嵌持久终端
-use-server-turn默认 true，自动拉取服务器 TURN 凭据并合并进 ICE
启动成功标志：日志出现 [Agent] Embedded terminal: ENABLED (shell=..., buffer=256KB)；Web 端鉴权后会出现「🖥️ 远程终端」卡片。
想用 TURN 中转？浏览器的 TURN 只来自 Agent 上报的 ice_servers。要么保持 -use-server-turn（配合服务器内嵌 TURN），要么给 Agent 配 -turn turn:host:3478 -turn-user U -turn-pass P。仅在信令机上跑 TURN 而 Agent 不上报，则没人能 relay。
STEP 3 · 客户端

③ 客户端接入

两种方式：浏览器控制台（零安装）或 CLI 客户端（端口映射 / 交互终端）。

🌐

方式 A · 浏览器控制台

1

登录

打开 http://host:8443/ 登录账户。

2

选择 Agent

列表里点你的 Agent，输入它的本地密码完成 P2P 鉴权。

3

使用

访问 allow_access 的端口，或点「🖥️ 远程终端 → 打开终端」用 xterm 终端。

⌨️

方式 B · CLI 客户端

端口映射 —— 把远端端口绑到本地：

client — 端口映射
$ ./bin/client -signal http://host:8443 \
    -username demo -user-password demo123 \
    -agent myagent -agent-password <pwd> \
    -map 127.0.0.1:18080=http
✓ mapping 127.0.0.1:18080 → agent:http

交互终端 —— Ctrl-] 退出（仅断开，不杀远端）：

client — 终端
$ ./bin/client -signal http://host:8443 \
    -username demo -user-password demo123 \
    -agent myagent -agent-password <pwd> -term
PS C:\>
🔒 四层安全：信令 token 限注册 → 账户登录拿 session → Agent 须带有效 owner-hash → DataChannel 建立后仍需 Agent 本地密码挑战-响应。Web 端只看得到 allow_access:true 的端口。
